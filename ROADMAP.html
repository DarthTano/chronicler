III

What content is yours to use

The single most important constraint for a D&D platform. Build on open content, let users supply the rest — and never copy from purchased books.

✓

SRD 5.1 / 5.2

Free to use

Core spells, monsters, items, classes and rules, released under Creative Commons. Your built-in baseline. Just credit Wizards of the Coast.

✓

User Homebrew

Free to use

Anything users create and type in themselves. If someone owns a book, they can hand-enter its content as their own homebrew — you give them the tools.

✕

D&D Beyond Books

Off limits

Buying a digital book grants viewing access, not redistribution rights. No API, no export, no scraping. Copying book content into your database is infringement.

IV

The toolkit

A stack chosen to get real-time, multi-user campaigns working without standing up a server farm.

Frontend
React

Component-driven UI that scales from a single sheet to a live campaign dashboard.

Backend & Data
Supabase

Postgres database, authentication, and real-time sync in one — the backbone for shared tables.

Real-time
Live Subscriptions

Initiative and HP changes push to every player at the table the instant they happen.

Intelligence
Claude API

Phase 5 — generate NPCs, encounters, and backstories on demand inside the tools.

V

Decisions locked in

What we've settled so far — the choices that shape everything downstream.

Who is the platform for? Players and DMs, equally
Solo tools, or live group play? Start solo, grow into live tables
Where does game content come from? SRD baseline + user homebrew
Can purchased D&D Beyond books be imported? No — not legally or technically
What's the visual direction? To be chosen from four themes
